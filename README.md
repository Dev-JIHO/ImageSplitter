# Image Splitter

큰 이미지를 여러 장의 A4 용지로 분할해 **인쇄용 PDF**로 내보내는 브라우저 전용 웹앱입니다. 인쇄한 A4들을 번호 순서대로 이어 붙이면 원하는 크기의 대형 포스터가 됩니다.

서버 업로드, 계정, 파일 저장, 프로젝트 히스토리가 전혀 없습니다. 모든 처리는 브라우저(Canvas)에서 이루어집니다.

## 주요 기능

- **두 가지 크기 지정 모드**
  - *A4 장수로 만들기(manual)*: A4 방향과 행·열 수를 직접 입력해 포스터 영역을 계산
  - *완성 크기로 만들기(target)*: '정확한 크기(가로+세로)'로 게시판 등 고정 크기를 그대로 지정하거나, '가로/세로 기준'으로 한 변만 입력(나머지는 이미지 비율). 최소 매수의 A4 격자를 자동 추천
- **이미지 배치**: 기본은 비율을 유지해 잘리지 않게 배치(contain). '꽉 채우기' 버튼으로 지정 영역을 가득 채우거나(넘침은 크롭), 확대 슬라이더·휠로 조절, 드래그로 위치 이동, '확대 원래대로'로 100% 복귀
- **풀칠 영역(overlap)**: 이웃한 페이지가 겹치도록 여백 탭을 두어 이어 붙이기 쉽게 함. 세로 이음새는 왼쪽, 가로 이음새는 위 페이지가 탭을 가지며, 탭이 없는 최하단·최우측 모서리 장에는 페이지 번호를 표시하지 않음
- **인쇄 배율 보정**: 접합 테스트 PDF(A4 2장)를 인쇄해 100mm 사각형의 실측값을 입력하면, 배율 조절이 안 되는 인쇄 앱에서도 실제 크기에 맞게 PDF를 보정
- **회전 / 확대 / 위치(crop focus) 조정**: 미리보기에서 드래그·휠·슬라이더로 조정
- **실시간 미리보기**: Canvas에 페이지 경계선, 프린터 여백, 풀칠 영역(빗금), 페이지 번호, 붙이는 순서를 표시
- **인쇄 옵션**: 프린터 여백(mm), 출력 해상도(150/200/300 DPI), 페이지 번호·경계선·풀칠 영역 표시 토글
- **반응형 UI**: 모바일에서는 설정/미리보기 패널 전환

## 기술 스택

- React + TypeScript
- Vite (개발 서버 / 번들러)
- jsPDF (PDF 생성)
- Vitest + jsdom (단위 테스트)

## 시작하기

요구 사항: Node.js (개발에 사용된 버전은 22.x)

```bash
npm install      # 의존성 설치
npm run dev      # 개발 서버 실행 (http://127.0.0.1:5173)
npm run build    # 타입 검사 후 프로덕션 빌드 (dist/)
npm run test     # 단위 테스트 1회 실행
npm run test:watch  # 테스트 watch 모드
```

## 사용 방법

1. (권장) 좌측 상단 **프린터 테스트**에서 접합 테스트 PDF(A4 2장)를 인쇄해 100mm 사각형 크기와 이음새를 확인합니다. 크기가 어긋나면 실측값을 입력해 배율을 보정합니다.
2. 이미지를 선택하거나 미리보기 영역에 끌어다 놓습니다. (JPG, PNG, WebP, GIF, AVIF 지원)
3. **A4 장수** 또는 **완성 크기** 모드를 선택하고 값을 입력합니다.
4. 미리보기에서 배치·회전·확대·위치를 조정합니다.
5. **PDF 내보내기**를 누르고 확인 모달에서 설정을 검토한 뒤 생성합니다.
6. 인쇄 시 **실제 크기(100%)**, **용지에 맞춤 끄기**로 인쇄합니다.
7. `1-1`부터 행 순서대로, 빗금(풀칠) 영역 위에 이웃 장을 겹쳐 붙입니다.

## 아키텍처 메모

도메인 로직은 `src/lib/`에 순수 함수로 분리되어 단위 테스트가 잘 갖춰져 있습니다. UI/상태는 `App.tsx`(조립) + `hooks`·`controls`·`preview`·`components`로 분해되어 있습니다. 분해 기록은 `docs/refactor-app-tsx.md`를 참고하세요.

## 측정 규칙

- A4 세로: 210mm × 297mm / A4 가로: 297mm × 210mm
- 기본 프린터 여백 5mm (가장자리 비인쇄 영역 보호)
- 좌표·치수는 mm 단위, PDF 출력 시 DPI로 픽셀 변환
